## Deployment & Automation (added 2026-07-14)

**Deployment pipeline:**
- Live site: axiongulf.com, deployed via Cloudflare Workers static assets (`wrangler.toml`, `[assets] directory = "./out"`), auto-deploying from the `main` branch of `nexoratech101/axiongulf` on GitHub. There is no GitHub Actions workflow in this repo — Cloudflare's own git integration handles the build (`npm run build` → `out/`) and deploy on every push to `main`.
- Standard change process: create a branch off latest `main` (`git fetch origin main && git checkout -B <branch> origin/main`), commit, push, open a PR (`gh`/GitHub MCP `create_pull_request`), then merge (squash) into `main` to trigger the live deploy. Confirmed working end-to-end on 2026-07-14 (PR #1 blog migration, PR #2 cadence docs).
- If a previous feature branch's PR has already been merged, don't stack new commits on the old branch — restart it from latest `main` (`git checkout -B <branch> origin/main`) per the standing convention, keeping the same branch name only if no unmerged work would be lost.
- **Known gotcha**: direct `git push` to a brand-new branch can intermittently 403 through the session's git proxy even when GitHub write access is fine — if this happens, wait a bit and retry once or twice before assuming it's a real permission problem (confirmed transient on 2026-07-14, second attempt succeeded).

**Axion Gulf blog system** (`src/lib/blog.ts`, `src/app/blog/page.tsx`, `src/app/blog/[slug]/page.tsx`):

**Known gotcha — Shopify's 25MP image limit (hit 2026-07-14)**: attaching a client-supplied photo directly to a Shopify article/product via `fileCreate` can silently fail if the source image exceeds 25 megapixels — the mutation itself returns no error, but the resulting `MediaImage` ends up with `fileStatus: FAILED` and `fileErrors: [{code: INVALID_IMAGE_RESOLUTION}]` on a follow-up query, so always check `fileStatus` after uploading, don't assume success from `fileCreate`'s immediate response. Fix: check `Image.open(path).size` (Pillow) before uploading; if width×height > ~20–22MP, downscale (e.g. `im.resize(...)`, `Image.LANCZOS`) to comfortably under the 25MP ceiling before running the staged-upload flow (`stagedUploadsCreate` → `curl -F` POST to the returned GCS URL with all returned `parameters` fields → `fileCreate` with the `resourceUrl` → poll `node(id){...on MediaImage{fileStatus, image{url}}}` until `READY`, then use that final `cdn.shopify.com` URL). This applies to any future image attached to a Shopify article via the cadence routine or manually.
